## 2. 核心知识点

### 2.1 虚拟内存的概念

**定义：** 虚拟内存是一种技术，允许进程不必完全加载到物理内存中就能执行。它将逻辑地址空间与物理地址空间分离。

**核心思想：**
- 进程的**逻辑地址空间**可以远大于**物理地址空间**
- 只有当前需要的部分才加载到物理内存
- 不需要的部分留在磁盘上

**虚拟内存的优势：**
1. 进程可以比物理内存更大
2. 更多进程可同时驻留在内存中（更高的多道程序设计度）
3. 加载进程的部分比加载整个进程快
4. 简化了内存管理

**实现方法：**
- 请求分页（Demand Paging）：最常见的方法
- 请求分段（Demand Segmentation）

### 2.2 请求分页

**原理：** 只在页面被访问时才将其加载到内存。

**页表扩展（增加几个关键位）：**
```
┌────────┬───┬───┬───┬───┬──────────┐
│ 有效位(P)│引用位│修改位│保护位│...  │ 页框号  │
└────────┴───┴───┴───┴───┴──────────┘
```
- P=1：页在内存中；P=0：页不在内存中（缺页）

**缺页中断（Page Fault）处理流程：**
```
1. 检查页表，发现有效位为0 → 触发缺页中断（陷入内核）
2. 操作系统查找空闲页框
3. 如果无空闲页框，执行页面置换
4. 从磁盘读取所需页面到页框中
5. 更新页表项
6. 重新执行导致缺页的指令
```

**有效访问时间（EAT）：**
```
EAT = (1 - p) × 内存访问时间 + p × 缺页处理时间

p = 缺页率

例如：内存访问=100ns, 缺页处理=25ms, p=1/1000
EAT = 0.999 × 100ns + 0.001 × 25,000,000ns
    ≈ 99.9ns + 25,000ns = 25,099ns

可见即使很小的缺页率也会显著影响性能！
```

### 2.3 页面置换算法

当发生缺页且无空闲页框时，需要选择一个页面换出。

#### 最佳置换（OPT — Optimal）
- 置换将来最长时间不会使用的页面
- **理论上最优，但无法实现（需要预知未来）**
- 用于比较其他算法的性能

#### 先进先出（FIFO）
- 置换最早进入内存的页面
- 简单实现（用队列）
- 缺点：可能置换经常使用的页面

**Belady异常：** FIFO算法中，增加页框数反而导致更多缺页！

#### 最近最少使用（LRU — Least Recently Used）
- 置换最长时间没有被访问的页面
- 基于局部性原理，效果接近OPT
- 实现方式：
  - 计数器：每次访问记录时间戳
  - 栈：将访问的页移到栈顶

#### 时钟算法（Clock / NRU — Not Recently Used）
- LRU的近似实现，也被称为**第二次机会算法**
- 使用引用位（R位）：
  - R=0：最近未被访问 → 可被置换
  - R=1：最近被访问过 → 清除R位，跳到下一页

```
循环扫描页框：
  if (引用位 == 0)
      置换此页
  else
      引用位 = 0
      继续扫描下一个
```

#### 增强的时钟算法
额外使用修改位（M位）：
- (0,0)：既未使用也未修改 → 最佳置换候选
- (0,1)：未使用但已修改 → 需要写回磁盘
- (1,0)：已使用但未修改 → 可能很快再次使用
- (1,1)：已使用且已修改 → 最差置换候选

**各算法对比：**
| 算法 | 性能 | 实现复杂度 | 特点 |
|------|------|-----------|------|
| OPT | 最优 | 不可实现 | 理论参考 |
| FIFO | 较差 | 简单 | Belady异常 |
| LRU | 很好 | 较高（需硬件支持） | 接近最优 |
| Clock | 较好 | 简单 | 实际系统常用 |

### 2.4 页框分配策略

**分配方式：**

| 维度 | 策略 | 描述 |
|------|------|------|
| **固定 vs 可变** | 固定分配 | 每个进程固定数量的页框 |
| | 可变分配 | 页框数根据进程需要动态变化 |
| **局部 vs 全局** | 局部置换 | 只能从自己的页框中置换 |
| | 全局置换 | 可以从任何进程的页框中置换 |

**常用组合：**
- 固定分配 + 局部置换（简单但不够灵活）
- 可变分配 + 全局置换（灵活但可能影响其他进程）
- 可变分配 + 局部置换（实际系统最常用）

**分配数量：**
- 最少页框数：确保进程能运行的最低页框数（取决于指令格式）
- 按比例分配：根据进程大小按比例分配
- 按优先级分配：给予高优先级进程更多页框

### 2.5 系统颠簸（Thrashing）

**定义：** 系统花在页面置换上的时间多于执行进程的时间。

**原因：** 运行的进程太多，每个进程分配的页框太少，频繁缺页。

**表现：**
- CPU利用率急剧下降
- 磁盘I/O大幅增加
- 系统吞吐量暴跌

**检测和控制颠簸：**

#### 工作集模型（Working Set Model）
- **工作集 W(t, Δ)：** 进程在最近Δ次内存访问中引用的页面集合
- 操作系统监控每个进程的工作集
- 总需求 = Σ工作集大小 ≤ 总可用页框数
- 如果总需求超过可用页框，挂起某些进程

#### 页面错误频率（PFF — Page Fault Frequency）
- 监控每个进程的缺页率
- 缺页率过高 → 增加页框分配
- 缺页率过低 → 减少页框分配，回收页框给其他进程

```
缺页率
  ↑
  ├──────────── 上限阈值：增加页框
  │
  │  ────────  正常范围
  │
  ├──────────── 下限阈值：减少页框
  └──────────→ 时间
```

### 2.6 其他考虑

**页大小选择：**
- 小页面：减少内部碎片，但页表更大
- 大页面：减少页表大小，但内部碎片多，I/O传输效率高
- 通常4KB或8KB（也有2MB的大页）

**预调页（Prepaging）：**
- 加载一个页时同时加载附近的页
- 利用空间局部性，减少初始缺页

**写时复制（Copy-on-Write）：**
- fork()时父子进程共享页面（标记为只读）
- 只有某个进程尝试修改时才真正复制
- 大大提高fork()的效率

## 5. 关键要点总结

- **虚拟内存**使进程逻辑空间独立于物理空间，可运行比内存大的程序
- **请求分页**按需加载页面，缺页时从磁盘调入
- **缺页率**对性能影响巨大：即使1/1000的缺页率也会使速度下降250倍
- **LRU**性能接近最优但实现复杂；**Clock算法**是实用的近似方案
- **全局分配+可变分配**是实际系统中最灵活的策略
- **颠簸**是虚拟内存系统最严重的问题，通过工作集或PFF控制
- **写时复制**是fork()高效实现的基石
