## 2. 核心知识点

### 2.1 内存管理的基本需求

| 需求 | 说明 |
|------|------|
| **重定位** | 程序可在内存任意位置执行，需将逻辑地址转为物理地址 |
| **保护** | 进程不能访问其他进程的内存空间 |
| **共享** | 多个进程可共享同一段内存（如共享库） |
| **逻辑组织** | 支持模块化编程，允许独立编译的模块 |
| **物理组织** | 管理主存和外存的存储层次 |

### 2.2 地址绑定

**地址转换的三个阶段：**

| 阶段 | 时机 | 特点 |
|------|------|------|
| **编译时** | 程序编译时 | 知道绝对地址，程序不能移动 |
| **加载时** | 程序装入内存时 | 加载器进行重定位 |
| **执行时** | 指令执行时 | 需要MMU硬件支持，灵活 |

**逻辑地址 vs 物理地址：**
- **逻辑地址（虚拟地址）：** 程序中使用的地址
- **物理地址（实地址）：** 内存硬件使用的地址
- **MMU（内存管理单元）：** 将逻辑地址转换为物理地址

### 2.3 内存分区

#### 固定分区（Fixed Partitioning）

```
┌────────────────┐ 8M
│   分区 4       │ ← 空闲
├────────────────┤ 6M
│   分区 3       │ ← 作业C（3MB，浪费1MB）
├────────────────┤ 4M
│   分区 2       │ ← 作业B（2MB，刚好）
├────────────────┤ 2M
│   分区 1       │ ← 操作系统
└────────────────┘ 0
```

- **内部碎片：** 分区内未使用的空间
- 简单但浪费严重

**分配算法（对大小相等的分区）：**
- 如果一个分区空闲且足够大，就分配

**分配算法（对大小不等的分区）：**
- 每个进程分配到能容纳它的最小分区

#### 动态分区（Dynamic Partitioning）

分区大小根据进程需求动态创建。

```
空闲分区链：根据分配算法不同而变化

首次适配（First Fit）：选择第一个足够大的空闲区
下次适配（Next Fit）：从上次分配位置开始搜索
最佳适配（Best Fit）：选择最小的足够大的空闲区
最差适配（Worst Fit）：选择最大的空闲区
```

**外部碎片：** 动态分区的主要问题——空闲空间被分割成小块

**紧凑/压缩（Compaction）：**
移动所有已分配分区，将空闲空间合并成一个大块
- 需要动态重定位能力
- 开销大（复制大量内存）

### 2.4 伙伴系统（Buddy System）

**原理：** 内存按2的幂次方分配，两个相邻的同等大小的块可以合并。

```
分配21KB → 分配32KB块（最接近的2的幂）
             ┌──────────────┐ 64K
             │    空闲      │
             ├──────────────┤ 32K
             │  已分配32K   │
             └──────────────┘ 0
```

**Buddy System 操作：**
1. 向上取到最接近的2的幂
2. 如果所需大小的块不存在，将更大的块一分为二
3. 释放时检查伙伴是否空闲，如果空闲则合并

### 2.5 分页（Paging）

**基本概念：**
- 将物理内存分成固定大小的**页框（Frame）**
- 将逻辑内存分成同样大小的**页（Page）**
- 页和页框大小相同（通常4KB）

**地址结构（32位系统，4KB页）：**
```
逻辑地址： ┌────────────┬──────────────┐
          │  页号 (20位) │ 偏移量 (12位) │
          └────────────┴──────────────┘
          可寻址2^20=1M页    每页4KB
```

**页表（Page Table）：**
- 每个进程有一个页表
- 页表[i] = 第i页对应的物理页框号
- 存储在内存中，PTBR（页表基址寄存器）指向页表

**地址转换过程：**
```
逻辑地址 = (p, d)    p: 页号, d: 页内偏移

1. 从页表中找到第p项 → 物理页框号 f
2. 物理地址 = f × 页大小 + d
```

**页表项（PTE）结构：**
```
┌────┬─────┬───┬───┬───┬──────────┐
│有效位│引用位│修改位│保护位│...   │ 页框号  │
└────┴─────┴───┴───┴───┴──────────┘
```
- 有效位(P)：该页是否在内存中
- 引用位(R)：该页是否被访问过
- 修改位(M)：该页在内存中是否被修改过（脏位）

**分页的优点：**
- 无外部碎片
- 每个页框都可分配
- 支持虚拟内存（配合第8章）

**分页的缺点：**
- 内部碎片（最后一页可能不满）
- 页表占用内存（对于大地址空间）
- 每次内存访问需两次访存（页表+数据）

**TLB（Translation Lookaside Buffer，快表）：**
- 高速缓存最近使用的页表项
- 减少页表访问开销

### 2.6 分段（Segmentation）

**原理：** 将程序按逻辑单元分段（代码段、数据段、栈段等）。

**段表项：**
```
┌──────────┬──────────┐
│  段基址   │  段界限   │
└──────────┴──────────┘
```

**逻辑地址：** (段号, 段内偏移)
- 段寄存器存放当前活动段的基址

**分段的优点：**
- 符合程序的逻辑结构
- 便于保护和共享（按段设置权限）
- 支持动态数据结构

**分段 vs 分页：**
| 特性 | 分页 | 分段 |
|------|------|------|
| 划分依据 | 固定大小（硬件决定） | 逻辑单元（程序员/编译器决定） |
| 对程序员可见 | 不可见 | 可见 |
| 碎片 | 内部碎片 | 外部碎片 |
| 大小 | 固定 | 可变 |

## 3. 重要术语

| 英文术语 | 中文翻译 | 说明 |
|---------|---------|------|
| MMU | 内存管理单元 | 负责地址转换的硬件 |
| Logical Address | 逻辑地址 | 程序使用的地址（虚拟地址） |
| Physical Address | 物理地址 | 实际内存硬件地址 |
| Internal Fragmentation | 内部碎片 | 分区内未使用的空间 |
| External Fragmentation | 外部碎片 | 分区之间的空闲空间 |
| Paging | 分页 | 将内存分为固定大小的页和页框 |
| Page Table | 页表 | 将逻辑页映射到物理页框的表格 |
| TLB | 快表/转换后备缓冲器 | 缓存页表项的高速存储器 |
| Segmentation | 分段 | 按逻辑单元划分内存 |
| Compaction | 紧凑/压缩 | 移动分区合并空闲空间 |
| Buddy System | 伙伴系统 | 基于2的幂次方的内存分配 |
| Frame | 页框/帧 | 物理内存中的固定大小块 |

## 5. 关键要点总结

- 内存管理需求：**重定位、保护、共享、逻辑组织、物理组织**
- **固定分区**产生内部碎片；**动态分区**产生外部碎片
- **伙伴系统**是折中方案，分配/释放速度快，但有内部碎片
- **分页**避免外部碎片，通过页表实现地址映射
- **TLB**是页表的高速缓存，对分页性能至关重要
- **分段**符合程序的逻辑结构，便于保护和共享
- 现代系统通常使用**段页式**（分段+分页）结合两者优势
