## 2. 核心知识点

### 2.1 计算机系统的基本组成

计算机系统由四个主要结构部件组成：

| 部件 | 功能 |
|------|------|
| **处理器（CPU）** | 控制计算机操作，执行数据处理功能 |
| **主存储器（内存）** | 存储数据和程序，易失性 |
| **I/O模块** | 在计算机和外部环境之间传输数据 |
| **系统总线** | 连接处理器、内存和I/O模块的通信通道 |

### 2.2 指令执行

**基本指令周期：**

```
取指阶段 → 执行阶段 → 取指阶段 → 执行阶段 → ...
```

- **取指阶段（Fetch）：** 处理器从内存中读取指令，PC（程序计数器）指向下一条指令
- **执行阶段（Execute）：** 处理器解释指令并执行所需操作

**指令分类：**
- 处理器-内存：在处理器和内存之间传送数据
- 处理器-I/O：在处理器和I/O模块之间传送数据
- 数据处理：对数据执行算术或逻辑操作
- 控制：改变指令执行顺序

### 2.3 中断机制

中断是现代操作系统实现多道程序设计的基础。

**中断的目的：** 提高处理器利用率，允许CPU在I/O操作期间执行其他任务。

**中断分类：**
1. **程序中断**：由指令执行产生的错误（除零、溢出等）
2. **时钟中断**：由处理器内部定时器产生，用于时间片调度
3. **I/O中断**：由I/O控制器产生，通知I/O操作完成
4. **硬件故障中断**：电源故障、内存奇偶校验错等

**中断处理流程：**
1. 设备发出中断信号
2. 处理器完成当前指令
3. 处理器检查中断信号
4. 保存当前程序状态（PSW + PC）
5. 根据中断向量跳转到中断处理程序
6. 执行中断处理程序
7. 恢复保存的程序状态
8. 继续执行被中断的程序

**多中断处理方式：**
- **顺序中断处理（禁止中断）：** 处理一个中断时禁止其他中断
- **嵌套中断处理（定义优先级）：** 允许高优先级中断打断低优先级中断处理

### 2.4 存储器层次结构

```
寄存器 ← 高速缓存(L1/L2/L3) ← 主存(RAM) ← 磁盘(SSD/HDD) ← 磁带
   ↑ 容量增大，速度降低，价格降低 ↑
```

**层次结构的设计依据：**
- **局部性原理（Locality of Reference）：**
  - 时间局部性：最近访问的数据很可能再次被访问
  - 空间局部性：被访问数据附近的数据很可能被访问

**存储层次的关键特性：**
| 层次 | 典型容量 | 典型访问时间 |
|------|---------|-------------|
| 寄存器 | ~1KB | ~1ns |
| L1 Cache | ~64KB | ~1-2ns |
| L2 Cache | ~256KB-1MB | ~5-10ns |
| L3 Cache | ~8MB-32MB | ~20-50ns |
| 主存 | ~8GB-64GB | ~50-100ns |
| SSD | ~256GB-2TB | ~10-100μs |
| HDD | ~1TB-10TB | ~5-10ms |

### 2.5 高速缓存（Cache）

**Cache的工作原理：**
- CPU首先在Cache中查找数据
- **命中（Hit）：** 数据在Cache中找到，快速返回
- **未命中（Miss）：** 数据不在Cache中，从下层存储加载整个块

**Cache映射方式：**
1. **直接映射：** 每个内存块只能映射到Cache中唯一的位置
2. **全相联映射：** 内存块可以映射到Cache中任意位置
3. **组相联映射：** 折中方案，每组包含多个Cache行

**替换策略（发生Cache未命中时）：**
- LRU（最近最少使用）
- FIFO（先进先出）
- 随机替换

**写入策略：**
- **写直达（Write-Through）：** 同时写入Cache和主存
- **写回（Write-Back）：** 只写入Cache，替换时再写回主存

## 3. 重要术语

| 英文术语 | 中文翻译 | 说明 |
|---------|---------|------|
| CPU | 中央处理器 | 执行指令、处理数据的核心部件 |
| PC (Program Counter) | 程序计数器 | 存放下一条要执行指令的地址 |
| IR (Instruction Register) | 指令寄存器 | 存放当前正在执行的指令 |
| MAR (Memory Address Register) | 存储器地址寄存器 | 存放要访问的内存地址 |
| MBR (Memory Buffer Register) | 存储器缓冲寄存器 | 暂存与内存交换的数据 |
| PSW (Program Status Word) | 程序状态字 | 包含条件码、中断使能位等信息 |
| Interrupt Vector | 中断向量 | 指向中断处理程序入口地址的指针 |
| DMA | 直接存储器访问 | 允许I/O设备直接访问内存而不经过CPU |
| Cache | 高速缓冲存储器 | CPU和主存之间的高速小容量存储器 |
| Locality of Reference | 局部性原理 | 程序访问数据的空间和时间聚集特性 |

## 5. 关键要点总结

- 计算机系统由**处理器、内存、I/O模块和系统总线**组成
- **中断机制**使CPU能在I/O等待期间处理其他任务，是多道程序设计的基础
- **存储器层次结构**基于局部性原理设计，从上到下容量递增、速度递减
- **Cache**利用局部性原理隐藏主存访问延迟，是现代计算机性能的关键
- DMA允许I/O设备**绕过CPU直接访问内存**，减少CPU干预
- 指令执行遵循**取指-执行**循环，程序计数器(PC)控制指令流
