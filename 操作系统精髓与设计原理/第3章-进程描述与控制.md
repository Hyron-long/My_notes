# 第3章 进程描述与控制

> Process Description and Control

---

## 1. 学习目标

- 理解进程的定义以及与程序的区别
- 掌握进程状态模型（两状态、五状态、七状态）
- 理解进程控制块（PCB）的结构和作用
- 掌握进程切换（上下文切换）的过程
- 理解操作系统控制进程的机制（创建、终止、挂起）
- 了解进程映像的组成

---

## 2. 核心知识点

### 2.1 什么是进程

**定义：** 进程是一个正在执行的程序实例，包括：
- **程序代码**（代码段，Text）
- **当前活动**（程序计数器、寄存器内容）
- **相关数据**（数据段、堆栈段）
- **系统资源**（打开的文件、分配的内存等）

**进程 vs 程序：**
| 方面 | 程序 | 进程 |
|------|------|------|
| 性质 | 静态的 | 动态的 |
| 存储 | 磁盘上的可执行文件 | 内存中的执行实体 |
| 生命周期 | 永久存在 | 暂时存在（创建→运行→终止） |
| 数量 | 一个程序可对应多个进程 | 每个进程对应一个可执行程序 |

### 2.2 进程状态模型

#### 两状态模型
```
非运行态 ←→ 运行态
```
- 简单但无法区分"就绪"和"阻塞"

#### 五状态模型（更重要）
```
       创建(New)
         ↓
       就绪(Ready) ←────────┐
         ↓                  │
       运行(Running) ──→ 阻塞(Blocked)
         ↓                  │
       终止(Terminated)     │
                            └── I/O完成 ──→ 就绪
```

**五种状态：**
| 状态 | 说明 |
|------|------|
| **新建(New)** | 进程刚被创建，尚未进入就绪队列 |
| **就绪(Ready)** | 进程已准备好运行，等待CPU分配 |
| **运行(Running)** | 进程正在CPU上执行 |
| **阻塞(Blocked)** | 进程等待某个事件（I/O完成、信号等） |
| **终止(Terminated)** | 进程执行完毕或被终止 |

**状态转换：**
- New → Ready：操作系统接纳进程，加入就绪队列
- Ready → Running：调度器选择进程分配CPU
- Running → Ready：时间片用完或被抢占
- Running → Blocked：等待事件（如I/O请求）
- Blocked → Ready：等待的事件发生
- Running → Terminated：进程执行完成或异常终止

#### 七状态模型（引入挂起）
增加两个状态：**挂起就绪（Suspend Ready）** 和 **挂起阻塞（Suspend Blocked）**

**挂起的原因：**
- 交换（Swapping）：将部分或全部进程从内存移到磁盘
- 用户请求：调试时暂停进程
- 父进程请求：父进程挂起子进程
- 操作系统原因：负载调节

### 2.3 进程控制块（PCB）

**PCB是进程存在的唯一标志**，包含以下信息：

| 类别 | 内容 |
|------|------|
| **进程标识** | PID、父进程PID、用户标识 |
| **处理器状态** | 通用寄存器、PC、PSW、栈指针 |
| **进程控制** | 进程状态、优先级、调度信息 |
| **内存管理** | 基址/界限寄存器、页表指针 |
| **I/O状态** | 打开文件列表、I/O请求队列 |
| **记账信息** | CPU时间、时钟、时间限制 |

### 2.4 进程切换（上下文切换）

**过程：**
1. 保存当前进程的处理器状态（寄存器、PC、PSW等）到PCB
2. 更新当前进程PCB中的状态信息
3. 将当前进程移到适当的队列
4. 选择另一个进程执行（调度）
5. 更新被选中的进程的PCB
6. 从被选中进程的PCB恢复处理器状态

**上下文切换的开销：**
- CPU时间（保存/恢复寄存器、更新数据结构）
- 切换时处理器空闲
- Cache/TLB刷新（可能）
- 频繁切换会严重降低性能

### 2.5 进程创建与终止

**进程创建的原因：**
- 新批处理作业提交
- 交互登录
- 操作系统创建服务进程
- 现有进程派生（fork）

**进程创建步骤：**
1. 分配唯一PID
2. 分配内存空间
3. 初始化PCB
4. 建立链接（放入就绪队列）
5. 创建或扩展其他数据结构

**进程终止的原因：**
- 正常完成
- 超时（超出时间限制）
- 内存不足
- 访问越界
- 保护错误
- 算术错误
- 父进程终止或请求终止子进程

### 2.6 进程映像

进程在内存中的布局（进程映像）：

```
高地址 ┌──────────────┐
       │    栈 (Stack)  │ ← 局部变量、函数调用帧
       │      ↓↓↓      │
       │               │
       │      ↑↑↑      │
       │   堆 (Heap)    │ ← 动态分配的内存
       ├──────────────┤
       │  数据段(Data)  │ ← 全局变量、静态变量
       ├──────────────┤
       │  代码段(Text)  │ ← 程序指令（只读）
低地址 └──────────────┘
```

## 3. 重要术语

| 英文术语 | 中文翻译 | 说明 |
|---------|---------|------|
| Process | 进程 | 正在执行的程序实例 |
| PCB | 进程控制块 | 记录进程信息的数据结构 |
| Context Switch | 上下文切换 | 进程间的CPU切换 |
| Process State | 进程状态 | 就绪/运行/阻塞等状态 |
| Dispatcher | 分派器 | 将CPU控制权交给选中进程 |
| Swapping | 交换 | 将进程在主存和辅存间移动 |
| Fork | 派生 | Unix创建子进程的系统调用 |
| Orphan Process | 孤儿进程 | 父进程已终止但子进程仍在运行 |
| Zombie Process | 僵尸进程 | 已终止但PCB仍存在的进程 |
| Process Image | 进程映像 | 进程在内存中的完整布局 |

## 5. 关键要点总结

- **进程**是正在执行的程序实例，包含了代码、数据和执行上下文
- **PCB**是进程在操作系统中的唯一标识，存储进程全部管理信息
- 进程的**五状态模型**：新建→就绪→运行→阻塞→终止
- **七状态模型**引入挂起状态，用于内存管理和负载调节
- **上下文切换**保存当前进程状态并恢复下一个进程状态，有较大性能开销
- **进程映像**包含代码段、数据段、堆和栈四个部分
- **fork()** 创建子进程，父子进程通过返回值区分（子进程返回0）
