## 2. 核心知识点

### 2.1 并发的基本概念

**并发（Concurrency）：** 多个进程/线程同时处于执行状态（不一定同时刻运行，可以交错执行）。

**并发的三种上下文：**
1. **多进程：** 多个独立进程同时运行
2. **多线程：** 一个进程内的多个线程并发执行
3. **中断处理：** 中断处理程序可能与正在执行的代码并发

**竞争条件（Race Condition）：**
- 多个进程/线程并发访问共享数据
- 最终结果取决于指令执行的具体顺序
- 导致结果不可预测和错误

### 2.2 临界区问题

**临界区（Critical Section）：** 访问共享资源（共享变量、文件、设备等）的代码段。

**临界区问题的三个要求：**
1. **互斥（Mutual Exclusion）：** 同时只能有一个进程在临界区中
2. **前进（Progress）：** 如果没有进程在临界区中，应允许申请者立即进入
3. **有限等待（Bounded Waiting）：** 进程申请进入临界区后，等待次数应有限

### 2.3 实现互斥的方法

#### 2.3.1 软件方法

**Dekker算法：** 第一个解决互斥问题的正确软件方案

**Peterson算法：** 更简洁的软件互斥方案
```c
int turn;          // 轮到谁
boolean flag[2];   // flag[i] = true 表示进程i想进入临界区

// 进程 i 的代码
flag[i] = true;
turn = j;                         // 谦让给对方
while (flag[j] && turn == j);     // 等待
// --- 临界区 ---
flag[i] = false;                  // 退出
```

#### 2.3.2 硬件方法

**中断禁用（单处理器）：**
```c
disable_interrupts();
// 临界区
enable_interrupts();
```
- 缺点：影响系统响应性，不适用于多处理器

**专用机器指令：**
- **Test-and-Set（TSL）：** 原子地读取并设置一个值
- **Compare-and-Swap（CAS）：** 比较值并原子地交换
- **Exchange（XCHG）：** 原子地交换两个值

```c
// Test-and-Set 实现自旋锁
boolean test_and_set(boolean *target) {
    boolean rv = *target;
    *target = true;
    return rv;
}

// 使用：
while (test_and_set(&lock));  // 获取锁（自旋）
// 临界区
lock = false;                 // 释放锁
```

#### 2.3.3 信号量（Semaphore）

信号量是一个整数变量，只能通过两个原子操作访问：

```c
// P操作（Wait / Down / Proberen）
wait(S) {
    while (S <= 0);  // 忙等
    S--;
}

// V操作（Signal / Up / Verhogen）
signal(S) {
    S++;
}
```

**信号量类型：**
| 类型 | 取值范围 | 用途 |
|------|---------|------|
| 二元信号量 | 0或1 | 互斥锁 |
| 计数信号量 | 非负整数 | 控制资源访问数量 |

**使用信号量解决生产者-消费者问题：**
```c
semaphore mutex = 1;   // 互斥访问缓冲区
semaphore empty = N;   // 空槽位数
semaphore full = 0;    // 满槽位数

// 生产者
wait(empty);
wait(mutex);
// 放入项目
signal(mutex);
signal(full);

// 消费者
wait(full);
wait(mutex);
// 取出项目
signal(mutex);
signal(empty);
```

#### 2.3.4 管程（Monitor）

管程是一种高级同步结构，包含：
- 共享数据结构
- 操作这些数据的过程（方法）
- 同步机制（条件变量）

```c
monitor ExampleMonitor {
    // 共享变量声明
    condition x, y;  // 条件变量

    procedure P1() {
        // ...
        x.wait();   // 等待条件x
        // ...
    }

    procedure P2() {
        // ...
        x.signal(); // 唤醒等待条件x的一个进程
        // ...
    }
}
```

**管程的特点：**
- 同一时刻只有一个进程可以在管程内执行
- 条件变量用于进程等待和唤醒
- 互斥由编译器/运行时自动保证

### 2.4 经典同步问题

#### 生产者-消费者问题（有界缓冲区）
- 生产者将数据放入缓冲区
- 消费者从缓冲区取出数据
- 缓冲区满时生产者等待，空时消费者等待

#### 读者-写者问题
- **读者优先：** 读者可同时读，写者需独占，读者优先于写者
- **写者优先：** 写者到达后新读者必须等待
- **公平方案：** 读者和写者按到达顺序服务

#### 哲学家就餐问题
- 五个哲学家围坐圆桌，五根筷子
- 每个哲学家需要两根筷子吃饭
- 问题：如何避免死锁？

## 3. 重要术语

| 英文术语 | 中文翻译 | 说明 |
|---------|---------|------|
| Race Condition | 竞争条件 | 多进程竞争共享数据导致结果不确定 |
| Critical Section | 临界区 | 访问共享资源的代码段 |
| Mutual Exclusion | 互斥 | 同时只有一个进程访问临界区 |
| Semaphore | 信号量 | 用于同步和互斥的整数变量 |
| Monitor | 管程 | 高级同步结构（共享数据+操作+条件变量） |
| Spinlock | 自旋锁 | 忙等待的锁（重复检查），适合短临界区 |
| Deadlock | 死锁 | 进程相互等待对方持有的资源 |
| Atomic Operation | 原子操作 | 不可中断的操作序列 |
| Condition Variable | 条件变量 | 管程中的等待/唤醒机制 |
| Bounded Waiting | 有限等待 | 等待进入临界区的次数有上限 |

## 5. 关键要点总结

- **竞争条件**是并发编程中最基本的问题，导致结果不确定
- **临界区**需要满足**互斥、前进、有限等待**三个条件
- **信号量**提供了一种简洁的同步原语：wait()递减，signal()递增
- **管程**将共享数据、操作和同步封装在一起，比信号量更安全
- **生产者-消费者**和**读者-写者**是最经典的同步问题
- 自旋锁适合**短临界区**；阻塞锁适合**长时间等待**
- 并发程序正确性验证要关注：**安全性（不会出错）** 和**活跃性（最终会进展）**
