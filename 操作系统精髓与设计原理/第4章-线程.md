## 2. 核心知识点

### 2.1 线程的概念

**定义：** 线程是进程内的一个执行路径（执行单元），是CPU调度的基本单位。

**进程与线程的关系：**
```
┌─────────────── 进程 ───────────────┐
│  ┌──── 线程1 ────┐  ┌──── 线程2 ────┐ │
│  │ 线程ID         │  │ 线程ID         │ │
│  │ 程序计数器(PC)  │  │ 程序计数器(PC)  │ │
│  │ 寄存器集       │  │ 寄存器集       │ │
│  │ 栈             │  │ 栈             │ │
│  └────────────────┘  └────────────────┘ │
│  ┌──────────────────────────────────────┐ │
│  │ 共享资源：代码段、数据段、打开文件等   │ │
│  └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

**线程 vs 进程：**
| 特性 | 进程 | 线程 |
|------|------|------|
| 资源拥有 | 拥有独立的地址空间和资源 | 共享进程的资源 |
| 调度 | 重量级，切换开销大 | 轻量级，切换开销小 |
| 通信 | IPC机制（管道、消息队列等） | 直接读写共享数据 |
| 创建/终止 | 开销大 | 开销小 |
| 独立性 | 独立实体 | 依赖进程存在 |

### 2.2 多线程的优势

1. **响应性：** 一个线程阻塞不影响其他线程（如GUI界面线程继续响应）
2. **资源共享：** 线程自动共享进程的内存和资源
3. **经济性：** 线程创建/切换开销远小于进程
4. **可扩展性：** 充分利用多核处理器，实现真正并行

### 2.3 用户级线程 vs 内核级线程

#### 用户级线程（ULT — User-Level Thread）
- 线程管理在用户空间完成，内核不知道线程的存在
- 优点：
  - 线程切换不需要内核模式特权（快）
  - 调度算法可以针对应用定制
  - 可在任何操作系统上运行
- 缺点：
  - 一个线程执行阻塞系统调用时，整个进程阻塞
  - 无法利用多处理器（内核只看到一个进程）

#### 内核级线程（KLT — Kernel-Level Thread）
- 线程管理由内核完成
- 优点：
  - 一个线程阻塞不影响其他线程
  - 可同时在多处理器上调度同一进程的多个线程
- 缺点：
  - 线程切换需要内核模式（较慢）

### 2.4 多线程模型

| 模型 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **多对一** | 多个ULT映射到单个KLT | 效率高，切换快 | 一个线程阻塞则全阻塞，不能并行 |
| **一对一** | 每个ULT映射到一个KLT | 真正的并行，线程独立 | 内核线程开销大，数量受限 |
| **多对多** | 多个ULT映射到相同或较少KLT | 兼顾并行和效率 | 实现复杂 |

```
多对一：  ULT ULT ULT ...    一对一：  ULT ULT ULT ...
           \  |  /                     |   |   |
            KLT                       KLT KLT KLT
             |                         |   |   |
            CPU                       CPU CPU CPU

多对多：  ULT ULT ULT ULT ...
           \    |     /
          KLT     KLT
            |       |
           CPU     CPU
```

### 2.5 线程库

**POSIX Pthreads（C语言）：**
```c
pthread_create(&thread, NULL, thread_func, arg);
pthread_join(thread, NULL);
pthread_mutex_lock(&mutex);
pthread_mutex_unlock(&mutex);
pthread_cond_wait(&cond, &mutex);
pthread_cond_signal(&cond);
```

**Windows API：**
```c
CreateThread(NULL, 0, ThreadFunc, arg, 0, &threadID);
WaitForSingleObject(thread, INFINITE);
```

**Java线程：**
```java
Thread t = new Thread(() -> { /* task */ });
t.start();
t.join();
```

### 2.6 线程管理

**基本线程操作：**
- **创建（Spawn/Start）：** 分配线程ID、寄存器、栈空间
- **阻塞（Block）：** 线程等待事件（锁、I/O、条件变量）
- **解除阻塞（Unblock）：** 等待的事件发生
- **结束（Finish）：** 线程完成任务（自愿终止）或异常终止

## 5. 关键要点总结

- **线程**是进程内的执行流，共享进程的地址空间和资源
- 线程切换开销远小于进程切换（不需要切换地址空间）
- **ULT**在用户空间管理，切换快但无法利用多核且阻塞会影响整个进程
- **KLT**由内核管理，可实现真正的并行但切换开销较大
- **多对多模型**是最佳折中方案，兼顾并行性和效率
- 多线程的主要优势：**响应性、资源共享、经济性、可扩展性**
- 常见的线程库：**Pthreads（POSIX）、Windows Threads、Java Threads**
