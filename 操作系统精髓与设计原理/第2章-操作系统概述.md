## 2. 核心知识点

### 2.1 操作系统的定义与目标

**定义：** 操作系统是控制应用程序执行、充当应用程序与计算机硬件之间接口的系统软件。

**三大目标：**
1. **方便性：** 使计算机更易于使用
2. **有效性：** 以高效的方式管理计算机资源
3. **扩展能力：** 允许有效地开发、测试和引入新功能

**操作系统作为资源管理器：**
- 管理处理器、内存、I/O设备、文件
- 在相互竞争的程序之间有序分配资源
- 跟踪资源使用状态、决定分配策略、分配/回收资源

### 2.2 操作系统的发展历史

| 阶段 | 时间 | 主要特征 |
|------|------|---------|
| **手工操作** | 1940s-1950s | 人工插拔线路、纸带输入 |
| **简单批处理** | 1950s | 监控程序（驻留程序）、作业自动过渡 |
| **多道批处理** | 1960s | 内存中同时驻留多个程序，CPU与I/O并行 |
| **分时系统** | 1960s-1970s | 多用户交互式使用，时间片轮转 |
| **现代操作系统** | 1980s-至今 | 图形界面、网络支持、安全、多核支持 |

**关键突破：**
- **多道程序设计：** 内存中同时存放多个作业，当当前作业等待I/O时，CPU切换到另一作业
- **分时系统：** 多个用户通过终端同时交互式访问系统，CPU按时间片轮转

### 2.3 操作系统的硬件支持

**受保护指令 vs 非特权指令：**
- 受保护指令（特权指令）：只能由操作系统在内核态执行
- 非特权指令：普通应用程序在用户态执行

**双模式操作：**
- **内核态（Kernel Mode）：** 可以执行所有指令，访问所有内存和硬件
- **用户态（User Mode）：** 只能执行非特权指令，内存访问受限

**模式切换机制：**
- 通过**系统调用（System Call）** 从用户态切换到内核态
- 通过**中断/异常**自动切换到内核态

**内存保护：**
- 基址寄存器和界限寄存器
- 防止用户程序访问操作系统内存或其他用户程序的内存

### 2.4 操作系统的主要成就

#### 2.4.1 进程（Process）
- 进程：正在执行的程序实例 = 程序代码 + 相关数据 + 进程控制块(PCB)
- 进程管理：创建/删除进程、进程同步、进程通信、死锁处理

#### 2.4.2 内存管理
- 内存分配与回收
- 地址转换（逻辑地址 → 物理地址）
- 虚拟内存技术
- 内存保护

#### 2.4.3 信息保护与安全
- 文件访问控制
- 用户认证（登录名+密码）
- 访问控制列表（ACL）

#### 2.4.4 调度与资源管理
- 处理器调度（长程/中程/短程调度）
- I/O调度
- 资源分配策略（公平性、效率、响应时间）

### 2.5 操作系统服务

**基本服务：**
- 程序创建和执行
- I/O操作访问
- 文件系统操作
- 通信（进程间通信、网络通信）
- 错误检测与处理

**系统调用（System Call）：**
应用程序访问操作系统服务的接口，例如：
- 进程控制：fork(), exec(), exit(), wait()
- 文件操作：open(), read(), write(), close()
- 设备操作：ioctl(), read(), write()
- 信息维护：getpid(), time(), gettimeofday()
- 通信：pipe(), socket(), shmget()

## 3. 操作系统结构

### 3.1 简单结构
- 无清晰分层
- 例如：MS-DOS（层次划分不明确，缺乏保护）

### 3.2 分层结构
- 操作系统分为多个层次
- 每层仅使用下层提供的服务
- 例如：THE操作系统

### 3.3 微内核结构
- 将最基本功能放入微内核
- 其他服务以用户态进程形式运行
- 优点：安全、可靠、易扩展
- 缺点：性能开销（进程间通信开销大）
- 例如：Mach, MINIX, QNX

### 3.4 模块化结构
- 面向对象的模块化设计
- 每个核心组件独立，通过接口通信
- 例如：Linux内核模块、Windows内核

### 3.5 虚拟机
- 在物理硬件上提供多个隔离的执行环境
- **Hypervisor (VMM):** 管理虚拟机的软件
- 例如：VMware, KVM, Xen, Hyper-V

## 4. 重要术语

| 英文术语 | 中文翻译 | 说明 |
|---------|---------|------|
| Kernel | 内核 | 操作系统的核心部分，常驻内存 |
| Multiprogramming | 多道程序设计 | 内存中同时驻留多个程序 |
| Time-Sharing | 分时系统 | 多用户交互式共享计算机 |
| System Call | 系统调用 | 用户程序请求操作系统服务的接口 |
| Kernel Mode | 内核态 | 可以执行特权指令的模式 |
| User Mode | 用户态 | 只能执行非特权指令的模式 |
| Process | 进程 | 正在执行的程序实例 |
| PCB | 进程控制块 | 存储进程信息的核心数据结构 |
| Virtual Memory | 虚拟内存 | 将磁盘当作内存扩展使用的技术 |
| Microkernel | 微内核 | 将操作系统服务最小化的设计 |

## 6. 关键要点总结

- 操作系统的三大目标：**方便、有效、可扩展**
- **多道程序设计** → CPU在I/O等待时切换任务，提高利用率
- **分时系统** → 多用户通过终端交互式共享计算机
- **进程** → 操作系统资源分配和调度的基本单位
- **双模式操作**（用户态/内核态）→ 保护操作系统免受用户程序破坏
- **系统调用**是用户程序与操作系统的唯一接口
- 内核设计有多种结构：**简单、分层、微内核、模块化**
