## 2. 核心知识点

### 2.1 调度器的类型

```
新作业 → [长程调度] → 就绪队列 → [短程调度] → CPU → 终止
                          ↑                    ↓
                        [中程调度]          I/O等待
                          ↓                    ↓
                       挂起队列 ←───→ I/O队列 → [I/O完成]
```

| 调度器 | 执行频率 | 功能 | 决策速度 |
|--------|---------|------|---------|
| **长程调度**（作业调度） | 低（分钟级） | 决定哪些作业进入系统 | 慢，可复杂 |
| **中程调度**（交换调度） | 中（秒级） | 决定哪些进程被挂起/换入 | 中 |
| **短程调度**（CPU调度） | 高（毫秒级） | 选择下一个运行的进程 | 必须极快 |

### 2.2 短程调度准则

**面向用户的准则：**
- **响应时间（Response Time）：** 从提交请求到开始响应的时间
- **周转时间（Turnaround Time）：** 从提交到完成的总时间
- **等待时间（Waiting Time）：** 在就绪队列中等待的总时间

**面向系统的准则：**
- **吞吐量（Throughput）：** 单位时间内完成的进程数
- **CPU利用率（CPU Utilization）：** CPU忙碌的时间百分比
- **公平性（Fairness）：** 各进程获得合理的CPU时间

**调度准则的权衡：**
- 响应时间 vs 吞吐量
- CPU利用率 vs 响应时间
- 公平性 vs 系统整体性能

### 2.3 调度算法

#### 先来先服务（FCFS — First Come First Served）
- 按照进程到达就绪队列的顺序分配CPU
- 非抢占式
- **护航效应（Convoy Effect）：** 长进程导致后续短进程长时间等待

```
示例：P1(24ms), P2(3ms), P3(3ms)
FCFS: |───P1(24)───|P2(3)|P3(3)|
平均等待时间 = (0 + 24 + 27) / 3 = 17ms
```

#### 最短作业优先（SJF — Shortest Job First）
- 选择预计CPU执行时间最短的进程
- 非抢占式版本：一旦开始执行直到完成
- **可证明是最优的**（最小化平均等待时间）
- 缺点：需要预知执行时间，可能导致长作业饥饿

```
同样示例：P1(24ms), P2(3ms), P3(3ms)
SJF: |P2(3)|P3(3)|───P1(24)───|
平均等待时间 = (0 + 3 + 6) / 3 = 3ms  ← 远优于FCFS!
```

#### 最短剩余时间优先（SRTF — Shortest Remaining Time First）
- SJF的抢占式版本
- 新到达进程的剩余时间更短 → 抢占当前进程

#### 轮转调度（RR — Round Robin）
- 每个进程获得一个**时间片（Time Quantum）**
- 时间片用完 → 放入就绪队列尾部
- 抢占式

```
时间片 q = 4ms, P1(24ms), P2(3ms), P3(3ms)
|P1(4)|P2(3)|P3(3)|P1(4)|P1(4)|P1(4)|P1(4)|P1(4)|
→ P2和P3快速完成，P1分多轮执行
```

**时间片的选择：**
- 太大 → 退化为FCFS
- 太小 → 上下文切换开销过大
- 经验值：时间片应远大于上下文切换时间（如10-100ms）

#### 优先级调度（Priority Scheduling）
- 每个进程分配优先级，高优先级的先执行
- 可抢占或非抢占
- **饥饿问题：** 低优先级进程可能永远得不到CPU
- **解决方案：老化（Aging）** — 等待时间越长，优先级逐渐提高

#### 多级队列调度（Multilevel Queue）
- 将就绪队列分为多个独立队列
- 每个队列有自己的调度算法
- 队列之间也需调度（通常是固定优先级抢占）

```
前台队列（交互式进程） → RR调度，优先级高
后台队列（批处理进程） → FCFS调度，优先级低
```

#### 多级反馈队列（Multilevel Feedback Queue）
- 允许进程在队列间移动
- 使用过多CPU的进程 → 降级到低优先级队列
- I/O密集型进程 → 保持在较高优先级
- 最通用的调度方案，现代操作系统广泛使用

```
优先级高 ┌──────────┐ q=8
        │ 队列 0   │ → RR，时间片小
        └────┬─────┘
             ↓ (时间片用完)
        ┌──────────┐ q=16
        │ 队列 1   │ → RR，时间片中
        └────┬─────┘
             ↓ (时间片用完)
        ┌──────────┐ 
        │ 队列 2   │ → FCFS（后台批处理）
        └──────────┘
优先级低
```

### 2.4 调度算法比较

| 算法 | 抢占 | 饥饿 | 优点 | 缺点 |
|------|------|------|------|------|
| FCFS | 否 | 否 | 简单，低开销 | 护航效应，交互差 |
| SJF | 否 | 可能 | 最小平均等待时间 | 需预知时间，长作业饥饿 |
| SRTF | 是 | 可能 | 最优平均等待 | 需预知时间，开销较大 |
| RR | 是 | 否 | 公平，适合交互 | 时间片难选择 |
| 优先级 | 可选 | 可能 | 灵活 | 低优先级饥饿 |
| 多级反馈队列 | 是 | 可用aging避免 | 自适应，通用 | 复杂 |

## 3. 重要术语

| 英文术语 | 中文翻译 | 说明 |
|---------|---------|------|
| Dispatcher | 分派器 | 将CPU交给选中进程的模块 |
| Turnaround Time | 周转时间 | 从提交到完成的时间 |
| Response Time | 响应时间 | 从提交到首次响应的时间 |
| Throughput | 吞吐量 | 单位时间完成的进程数 |
| Time Quantum | 时间片/时间量 | RR调度中每个进程的CPU时间 |
| Convoy Effect | 护航效应 | FCFS中短进程被长进程阻塞 |
| Aging | 老化 | 逐渐提高等待进程的优先级 |
| Preemption | 抢占 | 强制切换正在运行的进程 |
| Multilevel Feedback Queue | 多级反馈队列 | 根据CPU使用模式动态调整优先级 |

## 5. 关键要点总结

- **短程调度**是最频繁的调度，必须极快（毫秒级）
- **SJF/SRTF**在理论上最小化平均等待时间，但需要预知执行时间
- **轮转调度**适合交互式系统，关键在于选择适当的时间片
- **多级反馈队列**是实际系统最常用的方案，结合多种算法的优点
- 调度面临**响应时间、吞吐量、公平性**三者之间的权衡
- **老化(Aging)** 是防止优先级调度中饥饿的有效方法
- I/O密集型进程应优先于CPU密集型进程（更好的交互性）
