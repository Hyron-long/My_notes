## 2. 核心知识点

### 2.1 I/O设备分类

| 分类维度 | 类型 | 例子 |
|---------|------|------|
| **数据流向** | 输入设备 | 键盘、鼠标、传感器 |
| | 输出设备 | 显示器、打印机 |
| | 存储设备 | 磁盘、SSD、磁带 |
| **通信方式** | 块设备 | 磁盘（按块读写） |
| | 字符设备 | 键盘、串口（按字节/字符流） |
| **访问方式** | 顺序访问 | 磁带 |
| | 随机访问 | 磁盘、SSD |
| **传输速率** | 低速 | 键盘（~10B/s） |
| | 中速 | 磁盘（~100MB/s） |
| | 高速 | 网络（~10Gb/s） |

### 2.2 I/O组织技术

#### 程控I/O（Programmed I/O）
- CPU直接控制I/O操作
- CPU不断轮询设备状态寄存器
- 浪费CPU时间（忙等）

```
CPU: 发送命令 → 检查状态 → 未就绪 → 再检查 → 就绪 → 读/写数据 → ...
```

#### 中断驱动I/O（Interrupt-Driven I/O）
- CPU发送I/O命令后继续执行其他任务
- 设备完成操作后发送中断通知CPU
- 消除了忙等，但每次传输仍需CPU参与

```
CPU: 发送命令 → 继续其他工作
      ...
      收到中断 → 读/写数据 → 继续其他工作
```

#### 直接存储器访问（DMA — Direct Memory Access）
- 专门的DMA控制器处理数据传输
- CPU只需告知：设备地址、内存地址、传输字节数
- DMA完成后发中断通知CPU
- 适合大量数据传输

```
CPU: 配置DMA → 继续其他工作
DMA: 设备↔内存数据传输
DMA: 传输完成 → 中断CPU
CPU: 处理完成中断
```

**三种方式对比：**
| 方式 | CPU参与度 | 适用场景 |
|------|----------|---------|
| 程控I/O | 全部参与 | 极慢设备，简单系统 |
| 中断驱动I/O | 每次传输参与 | 中低速设备 |
| DMA | 仅初始化和完成中断 | 高速大块数据传输 |

### 2.3 I/O缓冲

**为什么需要缓冲？**
1. 速度匹配：CPU/内存和I/O设备的速度差异
2. 数据大小匹配：设备和应用程序的数据块大小可能不同
3. 支持写时复制语义

**缓冲方案：**

| 方案 | 描述 |
|------|------|
| **单缓冲** | 一个系统缓冲区，交替使用 |
| **双缓冲** | 两个缓冲区，一个传输时另一个处理 |
| **循环缓冲** | 多个缓冲区组成环形队列 |

```
双缓冲示例：
  时间 →
  缓冲区1: [写(磁盘)]────[读(用户)]────[写(磁盘)]────
  缓冲区2:        [读(用户)]────[写(磁盘)]────[读(用户)]
  → 读写完全并行，显著提高性能
```

### 2.4 磁盘调度

**磁盘访问时间：**
```
T_access = T_seek + T_rotation + T_transfer

T_seek（寻道时间）：磁头移动到目标柱面，约3-15ms（主要开销！）
T_rotation（旋转延迟）：扇区旋转到磁头下，平均值≈半圈，约2-8ms
T_transfer（传输时间）：数据读写时间，相对较小
```

**磁盘调度算法（按寻道时间优化排序）：**

#### 先来先服务（FCFS）
```
磁头初始位置：50
请求队列：95, 180, 34, 119, 11, 123, 62, 64

FCFS顺序：50→95→180→34→119→11→123→62→64
总寻道距离：45+85+146+85+108+112+61+2 = 644
```
- 简单公平，但寻道距离大

#### 最短寻道时间优先（SSTF）
```
请求：95, 180, 34, 119, 11, 123, 62, 64
SSTF：50→62→64→34→11→95→119→123→180
总寻道距离：12+2+30+23+84+24+4+57 = 236
```
- 性能优于FCFS，但可能导致饥饿（远处请求被无限延迟）

#### 扫描算法（SCAN / 电梯算法）
- 磁头向一个方向移动，处理沿途所有请求
- 到达最远端后反转方向

```
SCAN（从50向外(增大)方向）：
50→62→64→95→119→123→180→34→11
总寻道距离：12+2+31+24+4+57+146+23 = 299
```

#### C-SCAN（循环扫描）
- 磁头只向一个方向移动服务
- 到达最远端后快速返回到最远端（不服务）
- 提供更均匀的等待时间

#### LOOK 和 C-LOOK
- SCAN的改进版本
- 只移动到最后一个请求的位置就反转（而不是物理最远端）

**各算法对比：**
| 算法 | 性能 | 公平性 | 适用场景 |
|------|------|--------|---------|
| FCFS | 一般 | 最好 | 负载轻时 |
| SSTF | 较好 | 可能饥饿 | 不关键系统 |
| SCAN | 好 | 较好 | 中等负载 |
| C-SCAN | 好（均匀） | 好 | 高负载服务器 |
| C-LOOK | 很好 | 好 | 通用推荐 |

### 2.5 RAID（冗余磁盘阵列）

RAID使用多个磁盘提高性能和/或可靠性。

| RAID级别 | 描述 | 最少磁盘 | 容错能力 | 空间效率 |
|---------|------|---------|---------|---------|
| **RAID 0** | 数据条带化，无冗余 | 2 | 无 | 100% |
| **RAID 1** | 镜像 | 2 | 1块磁盘故障 | 50% |
| **RAID 2** | 位级条带化+汉明码 | 10+ | 1块磁盘故障 | 低（不常用） |
| **RAID 3** | 字节级条带化+专用奇偶校验盘 | 3 | 1块磁盘故障 | (n-1)/n |
| **RAID 4** | 块级条带化+专用奇偶校验盘 | 3 | 1块磁盘故障 | (n-1)/n |
| **RAID 5** | 块级条带化+分布式奇偶校验 | 3 | 1块磁盘故障 | (n-1)/n |
| **RAID 6** | 块级条带化+双分布式奇偶校验 | 4 | 2块磁盘故障 | (n-2)/n |
| **RAID 10** | RAID 1+0（先镜像再条带化） | 4 | 每组至少存活一个 | 50% |

**常用RAID级别：**
- **RAID 1**：最安全的简单方案，适合系统盘
- **RAID 5**：读写性能和容错能力的最佳平衡（最常用）
- **RAID 6**：比RAID 5多一层保护，适合大容量磁盘
- **RAID 10**：高性能+高可靠，成本高，适合数据库
- **RAID 0**：只追求性能，不推荐单独使用

### 2.6 磁盘缓存

位于主存中，缓存最近读取的磁盘扇区。

**替换策略：**
- LRU：最近最少使用
- LFU（Least Frequently Used）：最不经常使用
- 基于频率的替换

## 3. 重要术语

| 英文术语 | 中文翻译 | 说明 |
|---------|---------|------|
| DMA | 直接存储器访问 | 硬件直接传输数据，不经过CPU |
| Seek Time | 寻道时间 | 磁头移动到目标柱面的时间 |
| Rotational Delay | 旋转延迟 | 等待目标扇区旋转到磁头下 |
| SSTF | 最短寻道时间优先 | 选择最近的请求优先 |
| SCAN | 扫描/电梯算法 | 向一个方向扫描服务请求 |
| RAID | 冗余磁盘阵列 | 多磁盘组合提高性能/可靠性 |
| Stripping | 条带化 | 将数据分块分散在多磁盘上 |
| Parity | 奇偶校验 | 用于错误检测和恢复的冗余信息 |
| Disk Cache | 磁盘缓存 | 缓存最近磁盘数据的内存区域 |

## 5. 关键要点总结

- I/O控制方式演进：**程控I/O → 中断驱动I/O → DMA**
- **DMA**使CPU从数据传输中解放，适合高速大块传输
- 磁盘调度的关键是**减少寻道时间**（占访问时间的最大部分）
- **SCAN和C-LOOK**是实际系统中最常用的磁盘调度算法
- **RAID**用多磁盘平衡性能和可靠性：RAID 1（镜镜像）、RAID 5（分布式奇偶校验）、RAID 10（镜像+条带化）
- 现代SSD没有寻道时间，传统调度算法不再适用，需要新的调度策略
