# 第2章 关系模型

## 2.1 关系模型的数据结构及形式化定义

关系模型只有单一的数据结构——**关系**（二维表）。现实世界的实体及实体间的联系均用关系来表示。

### 2.1.1 关系

**定义2.1 域（Domain）**：一组具有相同数据类型的值的集合。

**定义2.2 笛卡尔积（Cartesian Product）**：
$$D_1 \times D_2 \times \cdots \times D_n = \{(d_1, d_2, \cdots, d_n) \mid d_i \in D_i, i=1,2,\cdots,n\}$$

- 每一个元素 $(d_1, d_2, \cdots, d_n)$ 叫一个 **n元组（元组）**
- 元组中的每一个值 $d_i$ 叫作一个 **分量**
- 域的基数：域中不同取值的个数。笛卡尔积的基数 $M = \prod_{i=1}^{n} m_i$

**定义2.3 关系（Relation）**：$D_1 \times D_2 \times \cdots \times D_n$ 的子集，表示为 $R(D_1, D_2, \cdots, D_n)$
- $n$ 是关系的 **目** 或 **度**（degree）
- $n=1$：一元关系；$n=2$：二元关系

**关系的三种类型**：基本关系（基表）、查询结果（临时表）、视图（虚表）。

**基本关系的6条性质**：
1. **列是同质的**：每列分量来自同一个域
2. **不同的列可出自同一个域**：必须取不同的属性名
3. **列的顺序无所谓**
4. **任意两个元组的码不能取相同的值**
5. **行的顺序无所谓**
6. **分量必须取原子值**：不可再分的数据项（规范化要求）

### 2.1.2 关系模式

**定义2.4 关系模式（Relation Schema）**：
$$R(U, D, DOM, F)$$

- $R$：关系名
- $U$：属性名集合
- $D$：属性所来自的域
- $DOM$：属性向域的映像集合
- $F$：属性间数据依赖关系的集合

**简化记法**：$R(U)$ 或 $R(A_1, A_2, \cdots, A_n)$

**关键术语**：
| 术语 | 含义 |
|------|------|
| 候选码 | 能唯一标识元组且真子集不能唯一标识元组的属性（组） |
| 主码 | 从多个候选码中选定的一个 |
| 主属性 | 候选码中的属性 |
| 非主属性 | 不包含在任何候选码中的属性 |
| 全码 | 关系模式的所有属性组成候选码 |

### 2.1.3 关系数据库

- **关系数据库模式**：所有关系模式的集合（类型）
- **关系数据库**：某时刻所有关系模式对应的关系的集合（值）

示例：学生选课数据库
- Student(Sno, Sname, Ssex, Sbirthdate, Smajor)
- Course(Cno, Cname, Ccredit, Cpno)
- SC(Sno, Cno, Grade, Semester, Teachingclass)

### 2.1.4 关系模型的存储结构

关系模型是逻辑结构，RDBMS负责设计和实现物理存储结构。

## 2.2 关系操作

### 2.2.1 基本的关系操作

- **查询操作**：选择、投影、连接、除、并、差、交、笛卡尔积
  - **5种基本操作**：选择、投影、并、差、笛卡尔积（其他操作可由这5种导出）
- **更新操作**：插入、删除、修改

**核心特点**：集合操作方式（一次一个集合），操作对象和结果都是关系。

### 2.2.2 关系数据语言的分类

```
关系数据语言
├── 关系代数（如ISBL）
├── 关系演算
│   ├── 元组关系演算（如ALPHA）
│   └── 域关系演算（如QBE）
└── 结构化查询语言（SQL）——兼具代数和演算双重特点
```

关系代数、元组关系演算和域关系演算三种语言在表达能力上是**等价**的（关系完备性）。

## 2.3 关系的完整性

### 2.3.1 实体完整性

**规则2.1**：若属性A是基本关系R的主属性，则A不能取空值（null）。

> 主码唯一标识实体，若主码为空则实体不可标识，与实体可区分性矛盾。

### 2.3.2 参照完整性

**定义2.5 外码（Foreign Key）**：设F是基本关系R的属性（组），但不是R的码，$K_s$ 是基本关系S的主码。若F与 $K_s$ 相对应，则F是R的外码。R为参照关系，S为被参照关系（目标关系）。

> R和S可以是同一个关系（自引用）。

**规则2.2**：若F是R的外码，与S的主码 $K_s$ 对应，则R中每个元组在F上的值必须：
- ① 取空值（F的每个属性值均为空）
- ② 等于S中某个元组的主码值

### 2.3.3 用户定义的完整性

针对具体应用的约束条件，反映特定语义要求。例如：属性取唯一值、非主属性不能取空值、取值范围限定等。

| 三类完整性 | 说明 |
|-----------|------|
| 实体完整性 | 主码不能为空，保证实体可区分 |
| 参照完整性 | 外码要么为空，要么等于被参照关系的主码值 |
| 用户定义的完整性 | 具体应用的语义约束 |

## 2.4 关系代数

### 运算符

| 类别 | 运算符 | 含义 |
|------|--------|------|
| 集合运算符 | $\cup$ | 并 |
| | $-$ | 差 |
| | $\cap$ | 交 |
| | $\times$ | 笛卡尔积 |
| 专门的关系运算符 | $\sigma$ | 选择 |
| | $\Pi$ | 投影 |
| | $\bowtie$ | 连接 |
| | $\div$ | 除 |
| 比较运算符 | $>, \ge, <, \le, =, \neq$ | |
| 逻辑运算符 | $\neg, \land, \lor$ | 非、与、或 |

### 2.4.1 传统的集合运算（二目运算）

设R和S具有相同的目n，属性取自同一个域：

| 运算 | 记号 | 定义 |
|------|------|------|
| 并 | $R \cup S$ | 属于R或属于S的元组 |
| 差 | $R - S$ | 属于R而不属于S的元组 |
| 交 | $R \cap S$ | 既属于R又属于S的元组（可用差表示：$R \cap S = R - (R - S)$） |
| 笛卡尔积 | $R \times S$ | R(n目, $k_1$个元组) × S(m目, $k_2$个元组) → (n+m)列, $k_1 \times k_2$行 |

**示例**：设关系R和S如下：

| R.A | R.B | R.C |
|-----|-----|-----|
| a | b | c |
| d | e | f |
| g | h | i |

| S.A | S.B | S.C |
|-----|-----|-----|
| d | e | f |
| j | k | l |

**$R \cup S$（并）**：

| A | B | C |
|---|---|---|
| a | b | c |
| d | e | f |
| g | h | i |
| j | k | l |

**$R - S$（差）**：

| A | B | C |
|---|---|---|
| a | b | c |
| g | h | i |

**$R \cap S$（交）**：

| A | B | C |
|---|---|---|
| d | e | f |
(只有d,e,f这个元组)

**$R \times S$（笛卡尔积）**：3×2=6个元组，4×2=8个元组(元组数)。

### 2.4.2 专门的关系运算

**记号约定**：
- $t[A_i]$：元组t中属性 $A_i$ 的分量
- $A = \{A_{i1}, A_{i2}, \cdots, A_{ik}\}$：属性列
- $\bar{A}$：去掉A后剩余的属性列
- **象集（Image Set）** $Z_x = \{t[Z] \mid t \in R, t[X] = x\}$：R中属性列X上值为x的诸元组在Z上分量的集合

#### 1. 选择（Selection） $\sigma_F(R)$

从行角度运算，选取满足条件F的元组。

$$\sigma_F(R) = \{t \mid t \in R \land F(t) = \text{'真'}\}$$

条件表达式基本形式：$X\theta Y$（$\theta \in \{>, \ge, <, \le, =, \neq\}$），可用 $\neg, \land, \lor$ 组合。

> **例2.4** 查询信息安全专业的全体学生。
> $$\sigma_{Smajor=\text{'信息安全'}}(Student)$$

> **例2.5** 查询2001年之后（含2001年）出生的学生。
> $$\sigma_{Sbirthdate \ge \text{'2001-1-1'}}(Student)$$

#### 2. 投影（Projection） $\Pi_A(R)$

从列角度运算，选取指定属性列组成新关系，会自动去重。

$$\Pi_A(R) = \{t[A] \mid t \in R\}$$

> **例2.6** 查询学生的学号和主修专业。
> $$\Pi_{Sno, Smajor}(Student)$$

| 20180005 | 信息管理与信息系统 |

#### 3. 连接（Join） $R \underset{A\theta B}{\bowtie} S$

从 $R \times S$ 中选取属性间满足 $\theta$ 条件的元组。

- **等值连接**：$\theta$ 为 "=" 的连接
- **自然连接**（$R \bowtie S$）：特殊等值连接，要求比较的分量为同名属性列，结果中去掉重复列
- **悬浮元组**：自然连接中被舍弃的元组
- **外连接**：保存悬浮元组，其他属性填NULL
  - **左外连接**：只保留R的悬浮元组
  - **右外连接**：只保留S的悬浮元组

**示例**（课本图2.8）：设关系R和S如下：

| R.A | R.B | R.C |
|-----|-----|-----|
| a | 5 | 3 |
| b | 6 | 7 |
| c | 8 | 10 |
| d | 12 | 2 |

| S.B | S.C | S.E |
|-----|-----|-----|
| 5 | 3 | 7 |
| 6 | 7 | 7 |
| 8 | 10 | 2 |
| 5 | 10 | 2 |
| 8 | 2 | 10 |

**$R \underset{R.C < S.E}{\bowtie} S$（非等值连接）**：从笛卡尔积中选出满足 $R.C < S.E$ 的元组（8个）。

**$R \underset{R.B = S.B}{\bowtie} S$（等值连接）**：从笛卡尔积中选出 $R.B = S.B$ 的元组，保留所有列（含重复）：

| R.A | R.B | R.C | S.B | S.C | S.E |
|-----|-----|-----|-----|-----|-----|
| a | 5 | 3 | 5 | 3 | 7 |
| b | 6 | 7 | 6 | 7 | 7 |
| c | 8 | 10 | 8 | 10 | 2 |
| c | 8 | 10 | 8 | 2 | 10 |

**$R \bowtie S$（自然连接）**：公共属性为B和C，去重后：

| A | B | C | E |
|---|---|---|---|
| a | 5 | 3 | 7 |
| b | 6 | 7 | 7 |
| c | 8 | 10 | 2 |

> R中(d,12,2)和S中(8,2,10)、(5,10,2)被舍弃，成为**悬浮元组**。

**外连接、左外连接、右外连接**结果为：

| A | B | C | E | | A | B | C | E | | A | B | C | E |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| a | 5 | 3 | 7 | | a | 5 | 3 | 7 | | a | 5 | 3 | 7 |
| b | 6 | 7 | 7 | | b | 6 | 7 | 7 | | b | 6 | 7 | 7 |
| c | 8 | 10 | 2 | | c | 8 | 10 | 2 | | c | 8 | 10 | 2 |
| d | 12 | 2 | NULL | | d | 12 | 2 | NULL | | NULL | 5 | 10 | 2 |
| NULL | 8 | 2 | 10 | | **(左外连接)** | | | | | NULL | 6 | 2 | 7 |
| NULL | 5 | 10 | 2 | | | | | | | **(右外连接)** | | | |
| **(外连接)** | | | | | | | | | | | | | |

#### 4. 除（Division） $R \div S$

$$R \div S = \{t_r[X] \mid t_r \in R \land \Pi_Y(S) \subseteq Y_x\}$$

$Y_x$ 为 $x = t_r[X]$ 在R中的象集。即：筛选出那些在R中象集**包含**了S在Y上投影的X值。

- **5种基本运算**：并、差、笛卡尔积、选择、投影
- 交、连接、除均可用基本运算表达

| a1 | b2 | c1 |
| a2 | b3 | c7 |
| a2 | b2 | c3 |

| a1 | b2 | c6 |

| S.B | S.C | S.D |
|-----|-----|-----|
| b1 | c2 | d1 |
| b2 | c1 | d1 |
| b2 | c3 | d2 |

- $\Pi_{B,C}(S) = \{(b1,c2), (b2,c1), (b2,c3)\}$

## 2.5 关系演算

关系演算以数理逻辑中的谓词演算为基础，按谓词变元不同分为元组关系演算和域关系演算。

### 2.5.1 元组关系演算语言 ALPHA

以元组变量作为谓词变元的基本对象。典型代表：E.F.Codd 提出的 ALPHA（INGRES的QUEL语言参照其设计）。

**6条语句**：GET、PUT、HOLD、UPDATE、DELETE、DROP

**基本格式**：`操作语句 工作空间名(表达式):操作条件`

**聚集函数**：

| 函数 | 功能 |
|------|------|
| COUNT | 对元组计数 |
| TOTAL | 求总和 |
| MAX | 求最大值 |
| MIN | 求最小值 |
| AVG | 求平均值 |

**元组变量的两个用途**：
1. 简化关系名
2. 操作条件中使用量词时必须用元组变量

**四种量词/逻辑检索方式**：
- 存在量词 $\exists$：存在满足条件的元组
- 全称量词 $\forall$：所有元组都满足条件
- 蕴涵 →：蕴含关系（如"选了X的全部课程"）
- 可用前束范式转换复杂表达式

**ALPHA 检索示例**：

> **简单检索**
> ```
> GET W(SC.Cno)                         -- 例2.14: 查询所有被选修的课程号
> GET W(Student)                        -- 例2.15: 查询全部学生数据
> ```

> **限定检索**
> ```
> GET W(Student.Sno, Student.Sbirthdate):
>     Student.Smajor='计算机科学与技术' ∧ Student.Sbirthdate < '2000-1-1'
> ```
> 例2.16: 查询计算机科学与技术专业2000年之前出生的学生的学号和出生日期。

> **带排序的检索**
> ```
> GET W(Student.Sno, Student.Sbirthdate):
>     Student.Smajor='计算机科学与技术' DOWN Student.Sbirthdate
> ```
> 例2.17: 查询结果按出生日期降序排序。

> **指定返回条数**
> ```
> GET W(1)(Student.Sno): Student.Smajor='信息安全'   -- 例2.18: 取出1个
> GET W(3)(SC.Sno, SC.Grade): SC.Cno='81003' DOWN SC.Grade  -- 例2.19: 成绩前3名
> ```

> **用存在量词 $\exists$ 的检索**
> ```
> RANGE SC X
> GET W(Student.Sname): ∃X(X.Sno=Student.Sno ∧ X.Cno='81002')
> ```
> 例2.21: 查询选修了81002号课程的学生姓名。

> **用全称量词 $\forall$ 的检索**
> ```
> RANGE SC SCX
> GET W(Student.Sname): ∀SCX(SCX.Sno≠Student.Sno ∨ SCX.Cno≠'81004')
> ```
> 例2.25: 查询不选81004号课程的学生姓名。也可用存在量词表达：
> ```
> GET W(Student.Sname): ¬∃SCX(SCX.Sno=Student.Sno ∧ SCX.Cno='81004')
> ```

> **同时使用 $\forall$ 和 $\exists$**
> ```
> RANGE Course CX
>       SC SCX
> GET W(Student.Sname): ∀CX ∃SCX(SCX.Sno=Student.Sno ∧ SCX.Cno=CX.Cno)
> ```
> 例2.26: 查询选修了全部课程的学生姓名。

> **用蕴涵（implication）的检索**
> ```
> RANGE Course CX
>       SC SCX
>       SC SCY
> GET W(Student.Sno): ∀CX(∃SCX(SCX.Sno='20180003' ∧ SCX.Cno=CX.Cno)
>                     → ∃SCY(SCY.Sno=Student.Sno ∧ SCY.Cno=CX.Cno))
> ```
> 例2.27: 查询至少选修了20180003号学生所选全部课程的学生学号。

> **聚集函数**
> ```
> GET W(COUNT(Student.Smajor))          -- 例2.28: 学生所选专业的数目
> GET W(AVG(SC.Grade)): SC.Cno='81003' ∧ SC.Semester='20202'
>                                        -- 例2.29: 平均成绩
> ```

**更新操作步骤**：
- **修改**：HOLD读取 → 宿主语言修改 → UPDATE送回（不允许修改主码）
- **插入**：宿主语言建新元组 → PUT插入
- **删除**：HOLD读取 → DELETE删除（注意参照完整性）

**更新操作示例**：

> **修改**（例2.30）
> ```
> HOLD W(Student.Sno, Student.Smajor): Student.Sno=20180004
> MOVE '信息管理与信息系统' TO W.Smajor
> UPDATE W
> ```
> 把学号20180004的学生转到信息管理与信息系统专业。

> **插入**（例2.31）
> ```
> MOVE '81009' TO W.Cno
> MOVE '计算机组织与结构' TO W.Cname
> MOVE '2' TO W.Ccredit
> MOVE '81005' TO W.Cpno
> PUT W(Course)
> ```

> **删除**（例2.32）
> ```
> HOLD W(Student): Student.Sno=20180007
> DELETE W
> ```

### 2.5.2 域关系演算语言 QBE

以域变量（元组的分量）作为谓词变元的基本对象。1975年由 M.M. Zloof 提出。

**核心特点**：基于屏幕表格的查询语言，用户通过填写表格构造查询，查询结果以表格形式显示，非常直观易用。

**操作框架**：

| 关系名 | 属性名 | ... |
|--------|--------|-----|
| 操作命令 | 元组属性值或查询条件或操作命令 | ... |

**主要操作符**：
| 操作符 | 含义 |
|--------|------|
| P. | 打印/显示 |
| U. | 修改 |
| I. | 插入 |
| D. | 删除 |

**关键要点**：
- **示例元素**：域变量，下面必须加下划线
- **"与"条件**：同行的条件为"与"；或不同行但示例元素值相同
- **"或"条件**：不同行且不同示例元素值
- **逻辑非**：运算符 $\neg$ 写在关系名下方
- **连接多个关系**：通过相同的连接属性值（示例元素值相同）
- **排序**：AO. 升序，DO. 降序，AO(i). / DO(i). 指定多列排序优先级

**QBE聚集函数**：CNT、SUM、AVG、MAX、MIN

**QBE 检索示例**：

> **简单查询**（例2.35）查询信息安全专业全体学生的姓名：

## 核心概念速记

| 概念 | 要点 |
|------|------|
| 域 | 同类型值的集合 |
| 笛卡尔积 | 域上所有可能组合 |
| 关系 | 笛卡尔积的有意义子集 |
| 候选码 | 能唯一标识元组的最小属性组 |
| 实体完整性 | 主码不能为空 |
| 参照完整性 | 外码 = 空 或 = 被参照关系主码值 |
| 5种基本运算 | 并、差、笛卡尔积、选择、投影 |
| 自然连接 vs 等值连接 | 自然连接去掉重复列 |
| 除运算 | 象集包含条件 |
| 外连接 | 保留悬浮元组 |
