# 第四章 网络层

## 4.1 网络层提供的两种服务

网络层提供的服务可以分为两大类思想：**面向连接的可靠服务**和**无连接的不可靠服务**。

### 4.1.1 虚电路服务（面向连接）

**虚电路（Virtual Circuit）** 是面向连接的服务，在通信之前需要先建立一条逻辑上的虚电路，然后所有分组都沿着这条虚电路按序传输。

| 特点     | 说明                         |
| -------- | ---------------------------- |
| 连接建立 | 需要先建立连接               |
| 分组携带 | 每个分组携带虚电路号         |
| 路由     | 固定的虚电路路径             |
| 顺序     | 分组按序到达                 |
| 可靠性   | 由网络层保证（如X.25协议）   |
| 差错处理 | 网络层负责差错控制和流量控制 |

### 4.1.2 数据报服务（无连接）

互联网采用的设计思路是**网络层向上只提供简单灵活、无连接的、尽最大努力交付的数据报服务**。

| 特点     | 说明                               |
| -------- | ---------------------------------- |
| 连接建立 | 不需要建立连接                     |
| 分组携带 | 每个分组携带完整的目的地址         |
| 路由     | 每个分组独立选择路由               |
| 顺序     | 分组可能失序、重复、丢失           |
| 可靠性   | 网络层不保证可靠交付，由运输层负责 |
| 差错处理 | 不负责差错控制和流量控制           |

**核心要点**：
- **数据报服务**：每个分组（IP数据报）独立发送，称为**数据报**
- **尽力而为（Best Effort）**：网络层尽最大努力交付，但不保证不丢失、不重复、不失序
- **端到端原则**：复杂的、端到端的可靠通信由运输层的TCP协议实现，网络层只需要提供简单服务即可

**虚电路服务与数据报服务的对比**：

| 对比项     | 虚电路服务           | 数据报服务               |
| ---------- | -------------------- | ------------------------ |
| 思路       | 可靠通信由网络层负责 | 可靠通信由运输层负责     |
| 连接的建立 | 必须有               | 不需要                   |
| 目的地址   | 每个分组有虚电路号   | 每个分组有完整的目的地址 |
| 路由选择   | 建立连接时选择，固定 | 每个分组独立选择         |
| 分组顺序   | 按序到达             | 可能失序                 |
| 差错控制   | 由网络层负责         | 由运输层负责             |
| 典型代表   | X.25、帧中继、ATM    | 互联网IP                 |

## 4.2 网际协议IP

**网际协议IP（Internet Protocol）** 是TCP/IP协议族中最核心的协议。与IP协议配套使用的还有三个协议：
- **ARP**：地址解析协议
- **ICMP**：网际控制报文协议
- **IGMP**：网际组管理协议

### 4.2.1 虚拟互连网络

#### 1. 中间设备

将网络互相连接起来需要使用的中间设备（中继系统）：

| 中继系统           | 工作层次     | 设备示例         |
| ------------------ | ------------ | ---------------- |
| 物理层中继系统     | 物理层       | 转发器（集线器） |
| 数据链路层中继系统 | 数据链路层   | 网桥（交换机）   |
| 网络层中继系统     | 网络层       | 路由器           |
| 高层中继系统       | 运输层及以上 | 网关             |

#### 2. 虚拟互连网络的意义

**互联网**可以在网络层使用**IP协议**，使多个异构的网络在IP层上"看起来"像一个统一的网络——**虚拟互连网络**（简称**IP网**）。

使用IP协议实现了**异构网络互连**：不同硬件类型、不同MAC帧格式的网络，在IP层上都能统一使用IP数据报进行通信。所有使用IP协议的设备都由IP层进行互联，路由器只处理IP数据报（分组），不关心下层的具体帧格式。

### 4.2.2 IP地址

#### 1. IP地址及其表示方法

**IP地址**是给互联网上的每一台主机（或路由器）的每一个接口分配一个在全世界范围内唯一的**32位标识符**。

IP地址由**ICANN（互联网名称与数字地址分配机构）** 进行分配。

**表示方法**：
- **点分十进制记法**：将32位地址每8位分为一组，转换为十进制，用点隔开
- 例如：`10000000 00001011 00000011 00011111` → `128.11.3.31`

#### 2. 分类的IP地址

**分类IP地址**将IP地址划分为若干个固定类，每一类地址由两个字段组成：

| 网络号 | 主机号  |
| ------ | ------- |
| net-id | host-id |

**各类地址的组成**：

| 类别    | 最高位 | 网络号长度    | 主机号长度 | 地址范围                    | 适用网络规模 |
| ------- | ------ | ------------- | ---------- | --------------------------- | ------------ |
| **A类** | 0      | 8位（1字节）  | 24位       | 1.0.0.0 ~ 127.255.255.255   | 大型网络     |
| **B类** | 10     | 16位（2字节） | 16位       | 128.0.0.0 ~ 191.255.255.255 | 中型网络     |
| **C类** | 110    | 24位（3字节） | 8位        | 192.0.0.0 ~ 223.255.255.255 | 小型网络     |
| **D类** | 1110   | —（组播地址） | —          | 224.0.0.0 ~ 239.255.255.255 | 多播         |
| **E类** | 1111   | —（保留地址） | —          | 240.0.0.0 ~ 255.255.255.255 | 保留         |

**特殊IP地址**：

| IP地址          | 类型        | 说明                             |
| --------------- | ----------- | -------------------------------- |
| 0.0.0.0         | —           | 本网络上的本主机                 |
| 127.x.x.x       | A类环回地址 | 本地环回测试，永远不出现在网络上 |
| 网络号全0       | —           | 本网络                           |
| 主机号全0       | —           | 网络地址，表示整个网络           |
| 主机号全1       | —           | 广播地址，发送给该网络上所有主机 |
| 255.255.255.255 | —           | 有限广播地址，仅限于本网络       |
| 169.254.x.x     | —           | DHCP失效时的链路本地地址         |

#### 3. 划分子网

从1985年起，在IP地址中增加"子网号字段"，即**两级IP地址 → 三级IP地址**：

**两级IP地址**：IP地址 ::= {<网络号>, <主机号>}
**三级IP地址**：IP地址 ::= {<网络号>, <子网号>, <主机号>}

**子网掩码（Subnet Mask）**：
- 子网掩码是一个32位二进制数，网络号和子网号部分为1，主机号部分为0
- 使用子网掩码可以从IP地址中提取网络号（包括子网号）
- 同一个网络中所有主机的子网掩码必须相同

**默认子网掩码**：

| 地址类别 | 默认子网掩码  | 二进制                              |
| -------- | ------------- | ----------------------------------- |
| A类      | 255.0.0.0     | 11111111 00000000 00000000 00000000 |
| B类      | 255.255.0.0   | 11111111 11111111 00000000 00000000 |
| C类      | 255.255.255.0 | 11111111 11111111 11111111 00000000 |

#### 4. 无分类编址CIDR

**CIDR（无类别域间路由选择，Classless Inter-Domain Routing）** 消除了传统的A、B、C类地址的概念。

**CIDR的特点**：
1. **消除分类**：不再区分A、B、C类，使用**网络前缀**取代网络号
2. **斜线记法**：IP地址 = {<网络前缀>, <主机号>}，写作"IP地址/前缀长度"
   - 例如：`128.14.35.7/20` 表示前20位是网络前缀
3. **地址块**：CIDR将网络前缀相同的连续IP地址组成一个**CIDR地址块**
   - 例如：`128.14.35.7/20` 的最小地址：`128.14.32.0`，最大地址：`128.14.47.255`
4. **路由聚合**（超网）：将多个地址块合并为一个更大的地址块，减少路由表条目

**CIDR地址块中的特殊地址**：
- 主机号全0的地址：网络前缀代表的网络
- 主机号全1的地址：广播地址

### 4.2.3 IP地址与MAC地址的区别

| 比较项   | IP地址                         | MAC地址                    |
| -------- | ------------------------------ | -------------------------- |
| 层次     | 网络层地址                     | 数据链路层地址             |
| 分配     | 由网络管理员分配               | 由硬件制造商固化           |
| 范围     | 全球唯一的逻辑地址             | 全球唯一的物理地址         |
| 可更改性 | 可以更改                       | 理论上不能更改（可虚拟）   |
| 作用     | 标识主机在网络中的位置（路由） | 标识设备在物理链路上的身份 |

**IP地址与MAC地址的关系**：
- IP地址是**逻辑地址**，指示主机在网络中的逻辑位置（类似于"收货地址"）
- MAC地址是**物理地址**，指示设备的物理身份（类似于"身份证号"）
- 在数据链路层传输时，需要将IP地址转换为MAC地址才能在同一链路上通信

### 4.2.4 地址解析协议ARP

**ARP（Address Resolution Protocol）** 用于将IP地址解析为MAC地址。

**ARP的工作原理**：
1. 每个主机（或路由器）都有一个**ARP缓存**，存储IP地址到MAC地址的映射
2. 当主机A要向同网络的主机B发送IP数据报时：
   - 先检查ARP缓存中是否有主机B的IP→MAC映射
   - 若无，则广播**ARP请求分组**（目的MAC地址为FF-FF-FF-FF-FF-FF）
3. 目标主机B收到请求后，发送**ARP响应分组**（单播）告知自己的MAC地址
4. 主机A收到响应后将映射存入ARP缓存

**ARP的四种典型情况**：
1. **发送方与接收方在同一网络**：直接ARP解析对方MAC地址
2. **发送方与接收方在不同网络**：ARP解析默认网关的MAC地址
3. **路由器转发**：路由器在转发时重新解析下一跳路由器的MAC地址

**注意**：ARP只解决同一局域网上IP地址与MAC地址的映射，不解决跨网络的问题。

### 4.2.5 IP数据报的格式

IP数据报由**首部**和**数据**两部分组成。首部前20字节是固定部分，后面是可选字段。

#### IP数据报首部格式（固定部分20字节）：

| 4位版本                      | 4位首部长度 | 8位区分服务    | 16位总长度 |
| ---------------------------- | ----------- | -------------- | ---------- |
| 16位标识                     | 3位标志     | 13位片偏移     |            |
| 8位生存时间TTL               | 8位协议     | 16位首部检验和 |            |
| 32位源IP地址                 |             |                |            |
| 32位目的IP地址               |             |                |            |
| 选项（长度可变，最多40字节） | 填充        |                |            |

**各字段详细说明**：

| 字段            | 长度 | 说明                                                         |
| --------------- | ---- | ------------------------------------------------------------ |
| **版本**        | 4位  | IP协议的版本，IPv4为4                                        |
| **首部长度**    | 4位  | 单位是4字节（32位字），最小5（20字节），最大15（60字节）     |
| **区分服务**    | 8位  | 旧称服务类型TOS，用于区分不同优先级的数据流                  |
| **总长度**      | 16位 | 首部+数据的总长度（字节），最大65535字节                     |
| **标识**        | 16位 | 用于分片重组，同一数据报的各分片具有相同标识                 |
| **标志**        | 3位  | 最高位保留（0）；DF（Don't Fragment）=1表示不分片；MF（More Fragment）=1表示后面还有分片 |
| **片偏移**      | 13位 | 分片在原数据报中的相对位置，单位是8字节                      |
| **生存时间TTL** | 8位  | 数据报在网络中可通过的路由器数的最大值，每经过一个路由器减1，为0时丢弃 |
| **协议**        | 8位  | 数据部分使用的协议类型（6-TCP，17-UDP，1-ICMP，2-IGMP）      |
| **首部检验和**  | 16位 | 只检验首部，不检验数据（每经过一个路由器重新计算）           |
| **源地址**      | 32位 | 发送方的IP地址                                               |
| **目的地址**    | 32位 | 接收方的IP地址                                               |
| **选项**        | 可变 | 用于支持排错、测量和安全等措施，很少使用                     |
| **填充**        | 可变 | 确保首部长度是4字节的整数倍                                  |

**IP数据报分片**：
- 当数据报长度超过网络MTU时，需要分片
- 分片可以在源主机或路由器上进行
- 重组只在目的主机上进行
- 分片偏移量必须是8字节的整数倍

### 4.2.6 IP层转发分组的流程

#### 1. 路由表

路由器中维护一个**路由表**，用于决定如何转发IP数据报。路由表的核心信息：

| 目的网络地址 | 子网掩码 | 下一跳地址 | 接口 |
| ------------ | -------- | ---------- | ---- |

#### 2. 分组转发算法

当路由器收到一个IP数据报时：
1. 提取目的IP地址D
2. 检查是否与路由器直接相连的网络匹配：
   - 用各网络的子网掩码和D逐位相与，看是否和相应网络地址匹配
   - 若匹配，则直接交付（不经过路由器）
3. 若未匹配，检查路由表中是否有**特定主机路由**（目的地址就是D）
4. 若未匹配，检查路由表中是否有**默认路由**（`0.0.0.0/0`）
5. 若都不匹配，则丢弃分组并报告转发差错

**最长前缀匹配**：当使用CIDR时，可能存在多个匹配项，选择**网络前缀最长**的那个（最具体的路由）。

## 4.3 网际控制报文协议ICMP

**ICMP（Internet Control Message Protocol）** 用于在IP主机和路由器之间传递控制信息和差错报告。

### 4.3.1 ICMP报文的种类

ICMP报文分为两大类：**ICMP差错报告报文**和**ICMP询问报文**。

#### 1. ICMP差错报告报文（5种）

| 类型               | 含义                       | 说明                                                 |
| ------------------ | -------------------------- | ---------------------------------------------------- |
| 终点不可达         | 路由器无法找到目的地址     | 包括网络不可达、主机不可达、协议不可达、端口不可达等 |
| 源点抑制           | 路由器因拥塞丢弃数据报     | 现已不使用（由TCP自行处理拥塞控制）                  |
| 时间超过           | TTL减到0                   | 可能原因：路由环路、分片在规定时间内未收齐           |
| 参数问题           | 首部字段值不正确           | 如首部长度错误                                       |
| 改变路由（重定向） | 路由器告诉主机有更好的路由 | 用于优化路由                                         |

**重要规则**：以下情况**不发送**ICMP差错报告报文：
- 对ICMP差错报告报文（避免无限循环）
- 对分片的第一片之后的所有分片
- 对具有多播地址的数据报
- 对具有特殊地址的数据报（如127.0.0.0或0.0.0.0）

#### 2. ICMP询问报文

| 类型            | 含义                    | 用途                 |
| --------------- | ----------------------- | -------------------- |
| 回送请求/回答   | Echo Request/Reply      | ping命令，测试连通性 |
| 时间戳请求/回答 | Timestamp Request/Reply | 同步时钟、测量时延   |

### 4.3.2 ICMP的应用举例

**1. ping（Packet InterNet Groper）**
- 使用ICMP回送请求和回答报文
- 测试主机之间的连通性
- 可计算往返时间RTT

**2. traceroute（路由跟踪）**
- Unix系统：traceroute；Windows系统：tracert
- 利用ICMP时间超过报文
- 逐步增加TTL值，依次获取路径上各路由器的信息

## 4.4 互联网的路由选择协议

### 4.4.1 路由选择协议的基本概念

**路由选择协议**的核心是**路由算法**，其目标是找到从源到目的的最优路径。

**理想路由算法的要求**：
- 正确性、完整性
- 简单性、低开销
- 自适应性（适应网络拓扑和通信量的变化）
- 稳定性、公平性、最优性

**两大类路由选择策略**：

| 策略         | 特点                                 | 优点               | 缺点             |
| ------------ | ------------------------------------ | ------------------ | ---------------- |
| **静态路由** | 管理员手动配置，不随网络变化自动调整 | 简单、可靠、开销小 | 不适应网络变化   |
| **动态路由** | 路由器通过路由协议自动学习路由       | 自动适应网络变化   | 开销大、配置复杂 |

### 4.4.2 分层次的路由选择协议

互联网采用**分层次的路由选择协议**，将整个互联网划分为许多较小的**自治系统AS（Autonomous System）**。

**自治系统AS**：在单一技术管理下的一组路由器，使用统一的路由选择协议和共同度量。

| 路由协议类型        | 作用范围 | 主要协议         | 说明           |
| ------------------- | -------- | ---------------- | -------------- |
| **内部网关协议IGP** | AS内部   | RIP、OSPF、IS-IS | 各AS可自行选择 |
| **外部网关协议EGP** | AS之间   | BGP-4            | 所有AS必须统一 |

```
┌─────────────────────────────────────────────────────┐
│                   互联网                             │
├───────────┬───────────┬───────────┬─────────────────┤
│    AS 1   │   AS 2    │   AS 3    │      ...        │
│  (RIP)    │  (OSPF)   │  (IS-IS)  │                 │
│     │         │          │        │                 │
│     └─────────┴──────────┴────────┘                 │
│                    BGP-4                            │
└─────────────────────────────────────────────────────┘
```

### 4.4.3 内部网关协议RIP

**RIP（Routing Information Protocol）** 是一种基于**距离向量**算法的路由协议。

#### 1. RIP的工作原理

- **距离定义**：到目的网络经过的路由器数（跳数），最大为15跳，16跳表示不可达
- 每隔30秒与相邻路由器交换**全部**路由信息
- 使用**距离向量算法**更新路由表

#### 2. 距离向量算法（Bellman-Ford）

当路由器收到邻居发来的路由表时：
1. 将收到的路由表"距离"字段加1，下一跳改为该邻居
2. 对每个目的网络：
   - 若原路由表中没有该网络 → 直接添加
   - 若有且下一跳是该邻居 → 替换为新距离（保持最新）
   - 若有但下一跳不是该邻居 → 若新距离更短则替换

#### 3. RIP的特点

| 优点         | 缺点                                         |
| ------------ | -------------------------------------------- |
| 实现简单     | 最大跳数15限制网络规模                       |
| 开销较小     | 路由更新可能引起"坏消息传播慢"（计数到无穷） |
| 适合小型网络 | 收敛慢，可能产生路由环路                     |

**防止路由环路的措施**：
- 水平分割（Split Horizon）：不向同一邻居发回从该邻居学到的路由
- 毒性逆转（Poison Reverse）：向邻居报告从该邻居学到的路由时，将其距离设为16（不可达）
- 触发更新：路由变化时立即发送更新，不等待定时器

#### 4. RIP报文的格式

RIP使用UDP协议，端口520。

### 4.4.4 内部网关协议OSPF

**OSPF（Open Shortest Path First，开放最短路径优先）** 是一种基于**链路状态**算法的路由协议。

#### 1. OSPF的工作原理

- 直接使用IP数据报传送（协议字段89）
- 路由器与相邻路由器建立**邻接关系**，交换链路状态信息
- 每个路由器根据收集到的链路状态，用**Dijkstra算法**计算到各网络的最短路径
- 链路状态更新只在网络发生变化时发送，使用**洪泛法**通知整个AS

#### 2. OSPF的特点

| 特点                 | 说明                             |
| -------------------- | -------------------------------- |
| 适应大规模网络       | 支持层次化划分（将AS划分为区域） |
| 收敛快               | 链路状态变化时快速更新           |
| 无路由环路           | 使用SPF算法保证无环              |
| 支持多条相同代价路径 | 可进行负载均衡                   |
| 支持类型服务路由     | 可根据TOS选择不同路径            |
| 支持验证             | 保证路由信息安全                 |
| 组播发送             | 使用组播地址224.0.0.5和224.0.0.6 |

#### 3. OSPF的分层结构

- **区域（Area）**：将一个AS划分为多个区域
- **主干区域（Area 0）**：连接各个区域的核心区域
- **区域边界路由器（ABR）**：连接不同区域
- **自治系统边界路由器（ASBR）**：连接其他AS

#### 4. OSPF的报文类型

| 类型 | 名称         | 用途                    |
| ---- | ------------ | ----------------------- |
| 1    | Hello        | 发现和维护邻居关系      |
| 2    | 数据库描述   | 同步链路状态数据库      |
| 3    | 链路状态请求 | 请求缺失的链路状态记录  |
| 4    | 链路状态更新 | 发送链路状态通告（LSA） |
| 5    | 链路状态确认 | 确认收到LSA             |

### 4.4.5 外部网关协议BGP

**BGP（Border Gateway Protocol，边界网关协议）** 是不同AS之间使用的路由协议。

#### 1. BGP的特点

- **不是找最佳路径，而是找"可达"路径**（因为AS之间的度量标准不同）
- **策略路由**：基于政治、经济、安全等策略决定路由
- BGP发言人之间通过TCP连接交换路由信息（端口179）
- 仅当路由发生变化时才更新

#### 2. BGP的工作原理

- BGP发言人：每个AS至少有一个路由器作为BGP发言人
- 建立TCP连接后成为**BGP对等体**
- 交换**路径向量**信息（经过的AS序列）
- 避免路由环路：BGP不接收包含自己AS号的路径

#### 3. BGP的报文类型

| 类型         | 用途                   |
| ------------ | ---------------------- |
| OPEN         | 建立邻居关系           |
| UPDATE       | 通告或撤销路由         |
| KEEPALIVE    | 保持连接（周期性发送） |
| NOTIFICATION | 报告错误               |

### 4.4.6 三种路由协议的比较

| 比较项   | RIP                 | OSPF                 | BGP                 |
| -------- | ------------------- | -------------------- | ------------------- |
| 类型     | 内部网关协议（IGP） | 内部网关协议（IGP）  | 外部网关协议（EGP） |
| 算法     | 距离向量            | 链路状态             | 路径向量            |
| 度量     | 跳数（≤15）         | 代价（带宽、时延等） | 策略（非数值度量）  |
| 交换内容 | 整个路由表          | 链路状态             | 可达路径（AS序列）  |
| 更新方式 | 周期性（30秒）      | 事件触发（变化时）   | 事件触发            |
| 收敛速度 | 慢                  | 快                   | 适中                |
| 适用规模 | 小型网络            | 大型网络             | 互联网骨干          |

## 4.5 路由器的构成

### 4.5.1 路由器的结构

路由器由两大部分组成：

```
┌─────────────────────────────────────────┐
│              路由器                       │
├───────────────────┬─────────────────────┤
│   路由选择部分     │     分组转发部分     │
│  （路由处理机）    │  ┌───────┬───────┐  │
│                   │  │输入端口│交换结构│  │
│  运行路由协议      │  ├───────┤       │  │
│  维护路由表        │  │输出端口│       │  │
│                   │  └───────┴───────┘  │
└───────────────────┴─────────────────────┘
```

1. **路由选择部分**：运行路由协议，构建路由表
2. **分组转发部分**：根据路由表转发数据报，包括：
   - **输入端口**：接收物理信号，转换为IP数据报，查表决定输出端口
   - **交换结构**：将分组从输入端口转移到输出端口
   - **输出端口**：缓存分组，转换为物理信号发送

### 4.5.2 交换结构

交换结构是路由器的核心，主要有三种实现方式：

| 方式             | 特点                 | 性能                     |
| ---------------- | -------------------- | ------------------------ |
| 通过存储器交换   | 早期路由器，CPU控制  | 速度慢，受限于存储器带宽 |
| 通过总线交换     | 数据报在总线上广播   | 所有端口共享总线带宽     |
| 通过互连网络交换 | 并行交换，如Crossbar | 性能最高，无阻塞         |

## 4.6 IPv6

### 4.6.1 IPv6的基本首部

IPv4地址空间即将耗尽，IPv6应运而生。

**IPv6的主要特点**：
- **地址长度**：128位（16字节），可提供巨大地址空间
- **简化的首部**：固定40字节，删除首部检验和等字段
- **更好的扩展性**：通过扩展首部支持新功能
- **流标签**：支持对特定流的区分处理
- **即插即用**：支持自动配置（无状态地址自动配置SLAAC）

**IPv6基本首部（40字节）**：

| 4位版本          | 8位通信量类   | 20位流标签  |
| ---------------- | ------------- | ----------- |
| 16位有效载荷长度 | 8位下一个首部 | 8位跳数限制 |
| 128位源地址      |               |             |
| 128位目的地址    |               |             |

### 4.6.2 IPv6的地址

**IPv6地址表示法**：
- **冒号十六进制记法**：每16位用十六进制表示，用冒号分隔
  - 例如：`6845:0000:0000:0000:0800:200C:00AB:1234`
- **零压缩**：连续的零段可用"::"表示（只能使用一次）
  - 例如：`6845::800:200C:AB:1234`
- **CIDR斜线记法**：`2001:0DB8::/32`

**IPv6地址类型**：

| 类型     | 说明                                       |
| -------- | ------------------------------------------ |
| **单播** | 一对一通信                                 |
| **多播** | 一对多通信（替代广播）                     |
| **任播** | 一对最近的一个通信（多个接口共享同一地址） |

**特殊IPv6地址**：
- `::/128`：未指定地址
- `::1/128`：环回地址
- `FF00::/8`：多播地址
- `FE80::/10`：链路本地单播地址

### 4.6.3 IPv4向IPv6的过渡

两种主要的过渡技术：

**1. 双协议栈（Dual Stack）**
- 路由器或主机同时运行IPv4和IPv6协议栈
- 根据目的地址选择使用IPv4或IPv6
- 需要同时配置IPv4和IPv6地址

**2. 隧道技术（Tunneling）**
- 将IPv6数据报封装在IPv4数据报中传输
- 适用于需要通过IPv4网络传输IPv6数据的场景
- 隧道两端需要双协议栈支持

## 4.7 IP多播

### 4.7.1 多播的基本概念

**IP多播**（也称**组播**）实现一对多的通信，发送方只需发送一份数据，只有加入该多播组的主机才会收到。

| 通信方式 | 特点                 | 效率                  |
| -------- | -------------------- | --------------------- |
| **单播** | 发送给一个特定主机   | 发送N份数据，浪费带宽 |
| **广播** | 发送给所有主机       | 打扰不需要的主机      |
| **多播** | 仅发送给加入组的主机 | 高效，只传一份        |

**IP多播地址**：采用D类地址（224.0.0.0 ~ 239.255.255.255）

### 4.7.2 网际组管理协议IGMP和多播路由选择协议

**IGMP（Internet Group Management Protocol）** 用于管理主机加入和离开多播组。

**IGMP的工作阶段**：
1. 当主机加入多播组时，向本网络多播路由器发送IGMP报文
2. 路由器周期性探询，了解有哪些组还有成员
3. 没有成员应答的组，路由器不再转发该组的多播数据

**多播路由选择协议**：在多播路由器之间建立多播转发树，常用的有：
- **距离向量多播路由协议DVMRP**
- **协议无关多播PIM**（密集模式和稀疏模式）

## 4.8 虚拟专用网VPN和网络地址转换NAT

### 4.8.1 虚拟专用网VPN

**VPN（Virtual Private Network）** 利用公共互联网作为传输介质，为专用网络建立安全的通信隧道。

**专用地址（私有IP地址）**：这些地址只能在内部网络使用，不能在互联网上路由：

| 地址类别 | 私有地址范围                               |
| -------- | ------------------------------------------ |
| A类      | 10.0.0.0 ~ 10.255.255.255（1个网络）       |
| B类      | 172.16.0.0 ~ 172.31.255.255（16个网络）    |
| C类      | 192.168.0.0 ~ 192.168.255.255（256个网络） |

**VPN的类型**：
- 内联网VPN：同一机构不同部门之间
- 外联网VPN：机构与合作伙伴之间
- 远程接入VPN：移动用户远程访问内部网络

### 4.8.2 网络地址转换NAT

**NAT（Network Address Translation）** 用于将私有IP地址转换为公有IP地址，以访问互联网。

**NAT的工作原理**：
1. 内部主机（私有IP）发送请求到外部
2. NAT路由器将源地址替换为公有IP，同时分配一个端口号
3. NAT路由器维护**转换表**，记录（私有IP+端口）与（公有IP+端口）的映射
4. 收到响应后，根据转换表将目的地址转换回私有IP

**NAPT（网络地址端口转换）**：最常用形式，使用一个公有IP地址和不同的端口号区分不同的内部主机。

**NAT的局限性**：
- 外部主机不能主动连接内部主机（除非手动配置端口映射）
- 某些协议（如FTP）在穿越NAT时可能出错

**NAT穿透技术**：
- 端口映射（Port Forwarding）
- UPnP（通用即插即用）
- STUN、TURN、ICE等协议
